What is SQL Injection (SQLi)?

SQLI or SQL Injection is a exploit that allows attackers to execute arbitrary SQL queries in databases that connect to the SQLI location. It happens when a database doesn't properly sanitise queries.

Mitigation:

Use parameterised queries, to prevent SQL injection. Other techniques such as filtering SQL commands fail as one could just use something like SELSELECETECT and the middle SELECT would be deleted but another would appear from the outside SEL...ECT

Famous Case:

2008 Heartland Payment Systems Breach: Using SQLi attackers got 130 million credit card numbers. SQLi allowed attackers to install malware to get the data.

Example of Basic Authentication Bypass:

Query example:

' OR '1'='1' --

This is a basic SQLi query used to bypass login authentication. Usually as the database looks something like

SELECT * FROM users WHERE username = 'user_input' AND password = 'pass_input';

If a user inputs

' OR '1'='1' --

as either the username or password, the query turns into:

SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = '';

Because '1'='1' is always true, this injection tricks the database into authenticating the user, if not sanitised appropriately.
